# CLAUDE.md — 큐닷 AX 과제 프로젝트 규칙

> 이 파일은 이 프로젝트에서 작업할 때 **항상 지켜야 할 핵심 규칙**이다.
> 작업 시작 전 PROGRESS.md로 현재 상태를 확인한다.

## 프로젝트 개요

브랜드 스토어 URL → 전 상품 크롤링 → AI 분석·구조화 → **큐닷 상품제안서(JSON)** 자동 정규화.
대상: 네이버 스마트스토어/브랜드스토어 + 비(非)네이버 공식몰. 목적은 **데이터 수집·AI 분석·정규화 역량 증명**(프론트는 최소).

## 핵심 원칙 (타협 금지)

1. **결정적 추출**: 화면 셀렉터 파싱이 아니라 **내부 JSON API**에서 추출한다. 셀렉터는 최후의 수단.
2. **룰 우선, LLM은 어려운 20%만**: 정형 데이터(가격·이미지·이름)는 코드로. 비정형 분류·요약(category/hashtags/usp)만 LLM. LLM에 전부 떠넘기지 않는다.
3. **AI 출력은 신뢰하지 않는다**: 모든 LLM 출력은 `validate.ts`의 스키마·범위 가드를 통과해야 한다. 7종 enum 밖 카테고리, 음수 가격 등은 차단.
4. **지어내지 않는다**: 크롤·AI로 못 얻는 값은 **공란(null) + 사유**(provenance). 추측으로 채우지 않는다.
5. **크롤링 매너/합법성**: 요청 간 딜레이(rateLimitMs) 유지. 과도한 요청 금지. "차단 우회 자랑"이 아니라 "리스크 인식"으로 접근. 인증은 사용자 본인 세션 재사용.

## 기술 스택

- 언어: **TypeScript** (큐닷 스택 친화, `tsx`로 실행)
- 크롤링: **Playwright + playwright-extra(stealth)** + **persistent 인증 세션**
- AI: **Claude / OpenAI structured output** (키 연동 시) — 현재는 룰 baseline
- 출력: JSON 파일 (`output/`). 웹(Next.js)은 선택/가산.

## 아키텍처 규칙

- **어댑터 패턴**: 스토어별 수집기는 `StoreAdapter` 인터페이스 구현. 새 몰 = 어댑터 1개 추가.
- 계층: `crawler`(통신) → `adapters`(수집) → `normalize`(매핑/검증) → `ai`(분류). 단방향 의존.
- **AI는 인터페이스(`Enricher`)로 추상화**. 룰/Claude/OpenAI를 교체 가능하게.
- 상품 단위 **에러 격리**: 하나 실패해도 로그 남기고 다음 상품 계속(전체 중단 금지).

## 성능 규칙

- **목록 우선**: 목록 API 1회로 얻는 필드는 상품마다 재호출하지 않는다.
- **가격은 배치**: product-benefits는 여러 상품을 묶어 호출(개별 호출 금지).
- 딜레이는 **줄이지 말 것**(차단 위험). 속도는 "호출 횟수 감소"로 확보.

## 보안 (절대 규칙)

- `naver-session/`(로그인 쿠키), `.env`(API 키)는 **절대 커밋 금지** (.gitignore 등록됨).
- 제출 repo에 개인 세션·키·신분정보가 들어가지 않게 확인.

## 작업 방식

- **응답은 한국어**로.
- 새 기능은 "1개 관통 → 확장" 순서. 한 번에 전체를 짜지 않는다.
- 작업 완료 후 **PROGRESS.md 갱신**.
- 커밋은 **작업이 일단락된 후 한꺼번에** (중간 커밋 없이). 커밋/푸시는 사용자가 요청할 때만.
